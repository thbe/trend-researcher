# Plan 04-03: `GET /api/topics/{id}` detail (Wave 3)

**Phase**: 4 ŌĆö Topic API & UI Shell
**Mode**: mvp
**Depends on**: Plan 04-02 (`v_topic_stats` access pattern + `routes/topics.py` exist)
**REQ-IDs in scope**: UI-002 (each row links to a topic detail route ŌĆö must return real data, not a 404 placeholder, per CONTEXT.md G7)

## Goal
Add `GET /api/topics/{topic_id}` returning a single topic with the list-endpoint fields PLUS `topic_metadata` and nested `sources` (ordered `observed_at DESC`). UUID auto-validated, 404 if not found. Phase 5+ extends this endpoint additively (no rewrite); Phase 6+ adds future `business_cases: []`.

## Slice shape (vertical)
HTTP route ŌåÆ handler ŌåÆ SELECT topic + JOIN v_topic_stats + SELECT topic_sources (ordered) ŌåÆ Pydantic response with nested list ŌåÆ JSON.

## Threat model (per ┬¦5.55)
- Read-only endpoint, same auth posture as the rest of `/api/*` (none, single-operator internal tool per ARC-004).
- `topic_id` typed as `UUID` in the path ŌĆö FastAPI auto-rejects malformed input with 422 before the handler runs. No SQL injection vector.
- 404 vs 200 distinguishes "topic does not exist" from "topic exists but has no sources" (latter is a valid 200 with `sources: []`).
- Response includes `topic_metadata` (JSONB blob captured per source-adapter ŌĆö see Topic model). Risk: if a source adapter ever caches a credential or PII payload there, it leaks via this endpoint. Mitigation: every existing source adapter writes only public payload fields (HN: id+score+by; NYT/Google: title+url+pubDate). No adapter writes secrets. Phase 5 may add a `topic_metadata` redaction allow-list if a new source needs it; not Phase 4.
- `sources[].url` is operator-clickable in the SPA ŌĆö `target=_blank rel=noopener` enforced in TopicDetail.vue (04-04 task).

## Must-haves (goal-backward verification)
- `GET /api/topics/{id}` returns real data when the topic exists ŌĆö **never** a 404 placeholder per CONTEXT.md G7 disposition.
