# Plan 04-05: api Dockerfile + StaticFiles mount + smoke + closeout (Wave 5)

**Phase**: 4 — Topic API & UI Shell
**Mode**: mvp
**Depends on**: Plan 04-04 (`web/` directory with package.json + buildable Vite app must exist)
**REQ-IDs in scope**: UI-001 (end-to-end SPA-deployable in compose; primary visible deliverable is "open http://localhost:8000/ in a browser")
**Autonomous**: false — final task T05 requires operator-driven smoke validation.

## Goal
Bind the four prior plans together into a single same-origin compose deliverable:
1. Multi-stage api Dockerfile (node builder → python runtime) that builds the SPA and embeds `dist/`.
2. FastAPI mounts `dist/` at `/` with `html=True` so SPA history-mode routes fall through to `index.html`.
3. Phase 4 smoke script.
4. README + SUMMARY closeout.

After this plan, `docker compose up -d --build api` brings up a container where `http://localhost:8000/` serves the Vuetify SPA and `http://localhost:8000/api/*` serves JSON.

## Slice shape (vertical)
Build (node + python in one image) → mount StaticFiles (after API routers) → smoke (assert SPA HTML + all 4 API endpoints) → docs.

## Threat model (per §5.55)
- Multi-stage build: node tooling stays in the builder stage — no `npm` / `node` / `node_modules` in the final python runtime image. Smaller attack surface, smaller image.
- `StaticFiles(directory=..., html=True)` serves `web/dist/`. Path-traversal: FastAPI's `StaticFiles` rejects `..` segments and absolute paths — pinned by FastAPI's own test suite; we don't re-test that. We DO test that `GET /` returns the SPA HTML and `GET /api/healthz` returns JSON (the catch-all does not steal API paths).
- `.dockerignore` excludes `web/node_modules/`, `web/dist/`, `**/.venv/`, `**/__pycache__/` so the build context stays small and we don't accidentally bake operator-local artifacts into the image.
- Same-origin (G2): no CORS, no cookies in scope, no auth changes. Posture unchanged from Phase 3.
- Compose binds api to `0.0.0.0:8000` (single-operator internal tool per ARC-004). No exposure to public networks is assumed; same posture as Phase 3.

<task id="T02" autonomous="true">
<read_first>
- services/api/src/api/main.py (current routers + lifespan from 04-02)
- services/api/src/api/dependencies.py (engine + session pattern; add WEB_DIST_DIR resolver here)
</read_first>
<action>
In `services/api/src/api/dependencies.py`:
- Add a module-level resolver:
  ```python
  import os
  from pathlib import Path

  def get_web_dist_dir() -> Path | None:
      """Resolve the path to the built SPA static files.

      In the container the multi-stage Dockerfile (Plan 04-05) sets
      WEB_DIST_DIR=/app/web_dist. In local pytest runs it is unset, so
      we return None and main.py skips the StaticFiles mount entirely.
      """
      raw = os.environ.get("WEB_DIST_DIR")
      if not raw:
          return None
      path = Path(raw)
      return path if path.is_dir() else None
  ```
- Add `get_web_dist_dir` to `__all__`.

In `services/api/src/api/main.py`:
- Import `StaticFiles` from `fastapi.staticfiles` and `get_web_dist_dir` from dependencies.
- After all `include_router(...)` calls (healthz, runs, topics), add:
  ```python
  _web_dist = get_web_dist_dir()
  if _web_dist is not None:
      app.mount("/", StaticFiles(directory=str(_web_dist), html=True), name="spa")
  ```
- The catch-all goes LAST — FastAPI evaluates routes in registration order, so `/api/*` routes always win.
- Update the module docstring to mention the SPA mount: "Phase 4 adds a catch-all StaticFiles mount at `/` serving the built Vuetify SPA from `WEB_DIST_DIR` (set by the multi-stage Dockerfile). API routes live under `/api/*` so the catch-all does not swallow them. The mount is skipped when `WEB_DIST_DIR` is unset or points to a missing directory — keeps `pytest` runs working without a built SPA."
</action>
<acceptance_criteria>
- `uv run --package api pytest -v services/api/tests/` exits 0 (no regressions; tests run without `WEB_DIST_DIR` set and the StaticFiles mount stays skipped).
- `WEB_DIST_DIR=$(pwd)/web/dist uv run --package api uvicorn api.main:app --port 8001` then `curl http://localhost:8001/` returns HTML containing `<div id="app">` (manual check, optional — primary check is in T04 smoke).
- `WEB_DIST_DIR=/nonexistent uv run --package api uvicorn api.main:app --port 8001` boots cleanly and `curl http://localhost:8001/` returns 404 (mount skipped gracefully) — pinned defensive behaviour.
</acceptance_criteria>

3. Trigger 1 crawler run to ensure there's at least one topic to detail (`docker compose run --rm crawler` — guarded so if it fails the smoke still attempts the API assertions; capture exit code but don't abort).
4. Assert `GET /api/healthz` → 200 + JSON with `status == "ok"`.
5. Assert `GET /api/runs?limit=5` → 200 + JSON has `runs` array.
6. Assert `GET /api/topics?sort=-breadth&limit=5` → 200 + JSON has `topics` array AND every row has `breadth` (number) + `longevity_seconds` (number). Use `jq -e '.topics | length > 0'` if pre-seeded; tolerate empty array but always assert response shape.
7. Assert sort whitelist: `GET /api/topics?sort=garbage` → 400.
8. Pick the first topic id from step 6 (if any), assert `GET /api/topics/{id}` → 200 + has `sources` array + `topic_metadata` field.
9. Assert `GET /` → 200 + Content-Type `text/html` + body contains `<div id="app">` (the SPA index).
10. Assert `GET /api/healthz` (after step 9) still returns 200 — proves the SPA catch-all did NOT steal `/api/*`.
11. On success: print `[smoke] phase 4 OK` and exit 0.
12. On any failure: print which assertion failed and exit 1.

Make it idempotent — re-runnable without state pollution.
`chmod +x scripts/smoke_phase4.sh`.
</action>
