## Goal
Bind the four prior plans into a single same-origin deliverable shaped for **Google Cloud Run with embedded Postgres + GCS-FUSE dump persistence**, following the `food-assistant` sibling-repo pattern verbatim. Specifically:

1. **3-stage Dockerfile** (node:20-alpine web-builder → uv-on-debian python-builder → ubuntu:24.04 runtime with postgresql-16 + postgresql-client-16) that builds the SPA, builds the Python venv, and ships embedded PG.
2. **docker-entrypoint.sh** port of food-assistant's 169-line bash: embedded-PG init / restore-from-dump chain / Alembic upgrade / DATABASE_URL toggle / SIGTERM trap → final dump + pg_ctl stop.
3. **scripts/pg-dump-rotate.sh** verbatim port from food-assistant with PG_DB + dump-path renames + flock + pg_restore --list verification + 3-slot atomic ring.
4. **FastAPI dump-debouncer middleware** that schedules `DB_DUMP_SCRIPT` 30 s after any successful write response (debounced, coalesced).
5. **FastAPI mounts `web/dist/` at `/`** with `html=True` (G2 same-origin, unchanged).
6. **Phase 4 smoke script** asserting both local-compose path (separate PG via DATABASE_URL) and prod-image path (embedded PG via empty DATABASE_URL).
7. **README + SUMMARY closeout.**

After this plan, `docker compose up -d --build api` brings up local compose (external PG, no dump-sync needed), AND `docker run --rm -p 8000:8000 -v $(pwd)/data:/app/data trend-api:dev` brings up the prod-shape image (embedded PG, restores from `/app/data/trend_researcher.dump` if present).

**Cloud Run deploy itself lands in 04-06** (via cloudbuild.yaml) — this plan ships the container.

## Slice shape (vertical)
Build (node + python + ubuntu+pg) → entrypoint + dump-rotate → middleware → mount StaticFiles → smoke (both modes) → docs.

## Threat model (per §5.55)
- **Embedded PG runs as `postgres` OS user** (initdb default). App connects via localhost md5. The `pg_hba.conf` shipped by the entrypoint is `local all all trust` (Unix socket only, for the entrypoint's own admin ops) + `host all all 127.0.0.1/32 md5` (TCP). No `0.0.0.0` listen. Surface = in-container only.
- **App role gets SUPERUSER** in the embedded-PG container only (single-tenant) — needed so `pg_restore --disable-triggers` works. Not a concern: nothing else lives in that PG instance, container boundary is the trust boundary.
- **Generated PG password** = `trend_embedded_$(hostname)` (per food-assistant). Not exported; only used inside container to construct DATABASE_URL. Not in logs.
- **Dump file at `/app/data/trend_researcher.dump`** is readable by container only. In prod the GCS bucket access is IAM-gated to the Cloud Run service account; no public-access policy on the bucket. (Documented in 04-06 cloudbuild.yaml.)
- **`pg_restore --list` verification** before promotion: a corrupt dump file (transient GCS write failure, partial upload, etc.) cannot rotate over `latest` — verify-then-promote. Restore chain `latest → .prev → fresh` gives one bad-dump tolerance.
- **`flock -n 9`** on lockfile: non-blocking. A stuck dump never queues another. Worst case = stale `.tmp` orphaned in `/tmp` (cleaned by container restart).
- **Final dump on SIGTERM** via `trap cleanup`. Cloud Run sends SIGTERM with 10 s default grace; we request 30 s grace in 04-06 cloudbuild.yaml `--timeout` / signal handling.
- **Multi-stage build**: node + uv + build tools stay in builder stages. Final ubuntu:24.04 runtime has only `postgresql-16`, `postgresql-client-16`, `ca-certificates`, the python venv, the web dist, and the entrypoint scripts. No node, no npm, no compiler. Smaller surface.
- **StaticFiles `html=True`**: FastAPI rejects `..` path traversal in its own tests; we don't re-test. We DO assert that `/api/*` still routes correctly after the catch-all mount (order matters — API routers registered first).
- **`.dockerignore`** excludes `web/node_modules`, `web/dist`, `**/.venv`, `**/__pycache__`, `.planning`, `.git`. Build context stays minimal; no operator-local artifacts baked in.
- **Compose**: api still binds `0.0.0.0:8000` (single-operator localhost; same posture as Phase 3). No exposure to public networks in local dev.
- **Local dev preserves Phase 3 posture**: separate `postgres:16-alpine` service in compose; entrypoint detects `DATABASE_URL` set → skips embedded boot. No GCS dependency for local dev.

```python
"""Debounced post-write pg_dump trigger middleware.

Schedules ``DB_DUMP_SCRIPT`` to run ``DB_DUMP_DEBOUNCE_MS`` after the
most recent successful write response. Coalesces bursts so a single
crawl run produces one dump, not N.

No-op if either env var is unset (local dev with external Postgres).
"""
from __future__ import annotations

import asyncio
import logging
import os
from typing import Awaitable, Callable

from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import Response

log = logging.getLogger(__name__)

_WRITE_METHODS = {"POST", "PUT", "PATCH", "DELETE"}


class DumpDebouncer:
    def __init__(self, script: str, debounce_ms: int) -> None:
        self._script = script
        self._debounce_s = debounce_ms / 1000.0
        self._pending: asyncio.Task[None] | None = None
        self._lock = asyncio.Lock()

    async def schedule(self) -> None:
        async with self._lock:
            if self._pending is not None and not self._pending.done():
                self._pending.cancel()
            self._pending = asyncio.create_task(self._run_after_delay())

    async def _run_after_delay(self) -> None:
        try:
            await asyncio.sleep(self._debounce_s)
        except asyncio.CancelledError:
            return
        try:
            proc = await asyncio.create_subprocess_exec(
                self._script,
                stdout=asyncio.subprocess.PIPE,
                stderr=asyncio.subprocess.PIPE,
            )
            stdout, stderr = await proc.communicate()
            if proc.returncode != 0:
                log.warning(
                    "dump-debouncer: script exit=%s stderr=%s",
                    proc.returncode, stderr.decode(errors="replace"),
                )
            else:
                log.info("dump-debouncer: completed")
        except Exception as exc:  # noqa: BLE001
            log.warning("dump-debouncer: failed: %s", exc)


class DumpDebouncerMiddleware(BaseHTTPMiddleware):
    def __init__(self, app, debouncer: DumpDebouncer) -> None:
        super().__init__(app)
        self._debouncer = debouncer

    async def dispatch(
        self, request: Request, call_next: Callable[[Request], Awaitable[Response]]
    ) -> Response:
        response = await call_next(request)
        if request.method in _WRITE_METHODS and 200 <= response.status_code < 400:
            await self._debouncer.schedule()
        return response


def build_dump_debouncer() -> DumpDebouncer | None:
    script = os.environ.get("DB_DUMP_SCRIPT")
    if not script:
        return None
    debounce_ms = int(os.environ.get("DB_DUMP_DEBOUNCE_MS", "30000"))
    return DumpDebouncer(script=script, debounce_ms=debounce_ms)
```
