# Plan 04-04: Vuetify 3 SPA scaffold (Wave 4)

**Phase**: 4 — Topic API & UI Shell
**Mode**: mvp
**Depends on**: Plan 04-03 (`GET /api/topics/{id}` must respond with real data for TopicDetail to render)
**REQ-IDs in scope**: UI-001 (TS + Vuetify SPA), UI-002 (topic browsing view, sortable by breadth/longevity/last_seen, row link → detail)

## Goal
Greenfield Vuetify 3 SPA at `web/` per CONTEXT.md G1/G3/G8 and `04-UI-SPEC.md`. After this plan, `cd web && npm install && npm run dev` opens a browser at `localhost:5173` (Vite proxies `/api/*` → `localhost:8000`) and shows TopicList (sortable v-data-table-server) + TopicDetail (back btn + metadata + sources). The compose-served same-origin build lands in Plan 04-05.

## Slice shape (vertical)
TypeScript fetch wrapper → typed `listTopics`/`getTopic` → Vue Router routes → Vuetify v-data-table-server (round-trips sort to API) + Vuetify v-card+v-list for detail → human-readable `formatLongevity`/`formatRelative`. End-to-end through the just-shipped `/api/topics*`.

## Threat model (per §5.55)
- SPA is single-operator internal tool, served same-origin (Plan 04-05). No CORS posture, no auth, no cookies, no localStorage credential storage.
- All API calls use relative `/api/...` paths through the fetch wrapper — no hardcoded hosts, no env-var leak surface.
- Source URLs from `TopicSource.url` are rendered as external links — MUST use `target="_blank" rel="noopener noreferrer"` to prevent tabnabbing and referrer leak.
- `topic.title`, `topic.description`, `source.url` interpolated into Vue templates use the default Vue `{{ }}` HTML-escaped binding — never `v-html`. Pinned by code-review checklist.
- `topic_metadata` rendered as `<pre>{{ JSON.stringify(topic_metadata, null, 2) }}</pre>` inside a `v-expansion-panel` — JSON.stringify escapes naturally, and the `<pre>` content is not eval'd.
- Vite dev proxy (`server.proxy`) only routes `/api/*` to localhost:8000 — does not expose the SPA dev server beyond localhost.
- npm dependencies pinned via `package-lock.json` committed to repo; npm audit baseline run at scaffold time and noted in `web/README.md`.

  - `const response = await fetch(path)`
  - If `!response.ok`: try `await response.json()` for `{detail}` → throw `ApiError(response.status, body.detail ?? response.statusText)`.
  - Return `await response.json() as T`.

Create `web/src/api/topics.ts`:
- Define TS types mirroring the Pydantic responses exactly:
  ```ts
  export interface Topic { id: string; title: string; description: string | null; first_seen_at: string; last_seen_at: string; observation_count: number; breadth: number; longevity_seconds: number }
  export interface TopicSource { id: string; source_name: string; url: string; native_rank: number; observed_at: string }
  export interface TopicDetail extends Topic { topic_metadata: Record<string, unknown>; sources: TopicSource[] }
  export interface TopicsListResponse { topics: Topic[]; limit: number; sort: string }
  ```
- Export `async function listTopics(params: { sort: string; limit: number }): Promise<TopicsListResponse>` building the query string with `URLSearchParams`.
- Export `async function getTopic(id: string): Promise<TopicDetail>` calling `request<TopicDetail>(`/api/topics/${encodeURIComponent(id)}`)`.

Create `web/src/lib/format.ts`:
- Export `function formatLongevity(seconds: number): string`:
  - `< 60` → `'<1m'`
  - `< 3600` → `${Math.floor(seconds/60)}m`
  - `< 86400` → `${Math.floor(seconds/3600)}h`
  - `>= 86400`: days = Math.floor(seconds/86400); hours = Math.floor((seconds % 86400)/3600); → `${days}d` if hours == 0 else `${days}d ${hours}h`
- Export `function formatRelative(iso: string): string`:
  - `const diff = (Date.now() - new Date(iso).getTime()) / 1000`
  - Same buckets as formatLongevity, suffix with `' ago'`. Future dates → `'just now'`.
- Pure functions; no Date mutation; no library deps.
