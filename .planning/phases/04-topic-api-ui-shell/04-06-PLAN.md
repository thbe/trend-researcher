**Phase**: 4 — Topic API & UI Shell

**Depends on**: Plan 04-05 (production container shape + entrypoint + dump-rotate + dump-debouncer all in place)
**REQ-IDs in scope**: UI-001 (closes the loop: the deployed URL the operator types is a real Cloud Run URL — local-compose was the dress rehearsal in 04-05; this plan ships the production endpoint, including the trigger surface that Cloud Scheduler will hit to keep data fresh)

## Goal
Replace the in-container `services/scheduler/` (alpine + crond + docker-cli + bind-mounted docker socket) with a PAT-secured HTTP endpoint that Cloud Scheduler can hit from outside the container, and ship the `cloudbuild.yaml` that deploys the Phase 4 image to Cloud Run with GCS-FUSE persistence + Secret Manager-mounted PAT.

Six concrete deliverables:

1. **PAT auth dependency** (`services/api/src/api/middleware/pat_auth.py`) — FastAPI `HTTPBearer` + `hmac.compare_digest`, **fail-closed** when env unset (503), 401 missing header, 403 wrong PAT.
2. **`POST /api/internal/crawl` endpoint** (`services/api/src/api/routes/internal.py`) — depends on PAT auth, fires `crawler.app.orchestrator.run_once` via `BackgroundTasks` (build sources + repos in the task, dispose engine after), returns **202 Accepted** immediately with `{"crawl_run_id": null, "status": "queued"}` (the actual run_id is written to `crawl_runs` by the orchestrator and observable via `GET /api/runs`).
3. **Workspace dep**: api depends on the crawler package so it can import `crawler.app.orchestrator` + `crawler.app.composition`.
4. **Scheduler deletion**: entire `services/scheduler/` tree gone; `scheduler:` block removed from `docker-compose.yml`; root `pyproject.toml` workspace `members` list updated (Phase 3 hot-fix `700533e` enumerated members — same enumeration shape, just minus the scheduler entry if it was listed; if not listed, no change there).
5. **`cloudbuild.yaml`** at repo root — port of food-assistant verbatim with trend-researcher renames + `--set-secrets=TREND_INTERNAL_PAT=trend-internal-pat:latest` + `--add-volume type=cloud-storage` mount.
6. **README + CLOUD-RUN-DEPLOY.md** — operator runbook for first deploy (Artifact Registry setup, GCS bucket setup, Secret Manager PAT seed, Cloud Build trigger, Cloud Scheduler job creation pointing at `/api/internal/crawl`).

After this plan, `gcloud builds submit --config=cloudbuild.yaml --region=europe-west2` deploys the full Phase 4 service to `https://trend-researcher-<hash>-ew.a.run.app/`, and a Cloud Scheduler job firing `POST /api/internal/crawl` with `Authorization: Bearer $PAT` triggers a fresh crawl.

## Threat model (per §5.55)

- **PAT is the ONLY auth surface that matters in v1.** Read endpoints stay public (single-operator localhost AND Cloud Run, which defaults to `--allow-unauthenticated` here for simplicity — operator can layer Cloud Run IAM later if they want belt-and-braces). The PAT gates only the write/trigger endpoint, which is the only place a leak gives an attacker anything they can't already get by curling read-only endpoints.
- **Constant-time compare** (`hmac.compare_digest`) — no timing-attack pivot from response latency to byte-prefix matching the PAT.
- **Fail-closed**: env var unset or empty string → **503**, NOT 401, NOT 200. The 503 carries a distinct error body so an operator hitting an unconfigured deployment instantly knows "Secret Manager mount missing" vs "wrong token". Critical: never default to a baked-in PAT, never auto-generate at boot (would silently work locally and then break in prod).
- **Bearer scheme** — `Authorization: Bearer <pat>`. `HTTPBearer(auto_error=False)` so the dep can distinguish "missing header" (401) from "wrong token" (403) from "service misconfigured" (503).
- **No PAT in logs.** The dep raises before any log emission of the header value. The internal endpoint logs only `crawl_run.queued` with no auth metadata. Cloud Run access logs strip Authorization by default.
- **PAT format** = `secrets.token_urlsafe(32)` → 44-char URL-safe base64 (~256 bits of entropy). Not guessable.
- **Rotation procedure**: `gcloud secrets versions add trend-internal-pat --data-file=<(echo -n "$NEW_PAT")` → `gcloud builds submit` (or just `gcloud run services update trend-researcher --set-secrets=...`) → update Cloud Scheduler job's auth header. No app-side rotation logic.
- **BackgroundTasks scope**: FastAPI's `BackgroundTasks` runs in the same process after response. A long crawl (currently ~10–30s for 3 sources × 100 items) blocks no responses but DOES hold the container alive past the response — fine because Cloud Run only scales down on idle, and the debouncer middleware (04-05) will trigger a final dump after the orchestrator's write completes.
- **Internal endpoint engine isolation**: the BackgroundTasks function builds its own SQLAlchemy engine via `crawler.app.composition.build_repository()` (CLI parity) and disposes it on completion. Does NOT reuse the api's request-scoped engine. Prevents pool exhaustion + event-loop confusion between request handler and background task.
- **No request body**: `POST /api/internal/crawl` accepts no body params in v1 (no `top_n` override, no `sources` filter). Operator changes `CRAWLER_TOP_N` / `CRAWLER_DISABLED_SOURCES` env via Cloud Run revision update, same shape as today. Keeps the trigger contract tiny and the attack surface single-dimensional.
- **Scheduler-deletion trust model improvement**: removing the docker-socket bind mount (`/var/run/docker.sock`) from compose **eliminates the largest trust hole in Phase 3** — the prior posture was "single-operator internal tool, docker socket trusted, root-equivalent". Local trigger now goes through the same PAT-gated HTTP path as prod (or operator runs `docker compose run --rm crawler run-once` directly, which never needed scheduler in the first place).
- **`.env` gitignore** — adds `.env` to root `.gitignore` so a local operator copy of the PAT cannot accidentally land in a commit. `.env.example` ships with PAT placeholder + generation snippet.
- **Cloud Run `--allow-unauthenticated`**: the read endpoints are intentionally public. If operator wants belt-and-braces later, swap to `--no-allow-unauthenticated` + grant `roles/run.invoker` to a specific identity (Cloud Scheduler service account for the trigger, IAP for the UI). Documented in CLOUD-RUN-DEPLOY.md but NOT implemented in v1.
- **GCS bucket access**: cloudbuild.yaml does NOT set bucket ACLs. Operator must create the bucket privately (`gcloud storage buckets create gs://trend-researcher-data --location=europe-west2 --uniform-bucket-level-access`) and grant `roles/storage.objectAdmin` only to the Cloud Run service account. CLOUD-RUN-DEPLOY.md spells this out as a prerequisite step.

```python
"""PAT (Personal Access Token) bearer-auth dependency for internal endpoints.

Used by /api/internal/crawl. Token lives in env var TREND_INTERNAL_PAT,
sourced from GCP Secret Manager in prod (Cloud Run --set-secrets) and from
a gitignored .env file in local dev.

Fail-closed semantics:
  - env unset or empty       -> 503 (service not configured)
  - Authorization header missing -> 401
  - token does not match     -> 403

Constant-time compare via hmac.compare_digest — no timing pivot.
"""
from __future__ import annotations

import hmac
import os

from fastapi import Depends, HTTPException, status
from fastapi.security import HTTPAuthorizationCredentials, HTTPBearer

_bearer = HTTPBearer(auto_error=False)

_ENV_VAR = "TREND_INTERNAL_PAT"


def _get_configured_pat() -> str | None:
    raw = os.environ.get(_ENV_VAR)
    if raw is None:
        return None
    raw = raw.strip()
    return raw or None


async def require_pat(
    creds: HTTPAuthorizationCredentials | None = Depends(_bearer),
) -> None:
    """Raise HTTPException unless the request carries the configured PAT."""

    configured = _get_configured_pat()
    if configured is None:
        raise HTTPException(
            status_code=status.HTTP_503_SERVICE_UNAVAILABLE,
            detail="internal-endpoint disabled: TREND_INTERNAL_PAT not configured",
        )
    if creds is None or not creds.credentials:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="missing bearer token",
            headers={"WWW-Authenticate": "Bearer"},
        )
    if creds.scheme.lower() != "bearer":
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="bearer scheme required",
            headers={"WWW-Authenticate": "Bearer"},
        )
    if not hmac.compare_digest(creds.credentials.encode(), configured.encode()):
        raise HTTPException(
            status_code=status.HTTP_403_FORBIDDEN,
            detail="invalid token",
        )


__all__ = ["require_pat"]
```

```python
"""Internal trigger endpoints. PAT-gated.

POST /api/internal/crawl — fires crawler.app.orchestrator.run_once in a
BackgroundTask and returns 202 Accepted. Replaces the deleted
services/scheduler/ crond container; Cloud Scheduler hits this in prod.
"""
from __future__ import annotations

import os

import structlog
from fastapi import APIRouter, BackgroundTasks, Depends, status

from api.middleware.pat_auth import require_pat

_log = structlog.get_logger(__name__)

router = APIRouter(prefix="/api/internal", tags=["internal"])


async def _run_crawl_isolated() -> None:
    """Build a fresh engine + repos, run the crawler, dispose.

    Mirrors the crawler CLI's `_main()` pattern (cli.py) so the engine
    pool binds to the FastAPI event loop the BackgroundTask runs on, and
    is disposed cleanly even if `run_once` raises.
    """
    from crawler.app.composition import build_repository, build_sources
    from crawler.app.orchestrator import run_once as run_once_async

    top_n = int(os.environ.get("CRAWLER_TOP_N", "100"))
    sources = build_sources()
    topic_repo, crawl_run_repo, engine = build_repository()
    try:
        stats = await run_once_async(sources, topic_repo, crawl_run_repo, top_n)
        _log.info(
            "internal.crawl.completed",
            crawl_run_id=stats.get("crawl_run_id"),
            totals=stats.get("totals"),
        )
    except Exception as exc:  # noqa: BLE001 — background context, must not crash worker
        _log.exception("internal.crawl.failed", error=str(exc))
    finally:
        await engine.dispose()


@router.post(
    "/crawl",
    status_code=status.HTTP_202_ACCEPTED,
    dependencies=[Depends(require_pat)],
)
async def trigger_crawl(background_tasks: BackgroundTasks) -> dict[str, str]:
    """Queue a one-shot crawl. Returns immediately; observable via /api/runs."""
    background_tasks.add_task(_run_crawl_isolated)
    _log.info("internal.crawl.queued")
    return {"status": "queued"}


__all__ = ["router"]
```
