## Key decisions executed (from PLAN frontmatter)

- **D-Q1 — First-non-empty merge for description**: COALESCE in the repo's update path. Earliest non-empty wins. Matches the backfill script's `DISTINCT ON … ORDER BY observed_at ASC` semantic so re-crawl post-backfill is stable.
- **D-Q2 — One-shot backfill script**: `scripts/backfill_descriptions.py`, async, two passes, both `WHERE … IS NULL`-guarded. Not a separate Cloud Run job — operator runs it locally once against the prod DSN, or it'll happen naturally as re-crawls fill in (description-side only — resolved_url backfill is doomed-by-design for the 81 modern opaque tokens).
- **D-Q3 — Hybrid decoder + structlog.warn + fallback to original URL**: implemented exactly. Live finding: **all 81 prod CBM tokens are modern opaque format with no embedded URL** — they decode to internal Google IDs (`AUNF`/`AUUF`-prefixed payloads). The documented fallback path (`resolved_url` stays NULL, SPA uses `url`) is the load-bearing behaviour in production today. Old-style decoding (3 synthetic tests pass) is dead-code-shaped insurance against Google reverting their format.
- **D-Q4 — `topic_sources.resolved_url TEXT NULL`**: additive, no existing-row migration. Read path uses `resolved_url ?? url` (SPA) and JSON serialises `null` for non-google_news rows.
- **D-Q5 — SPA shape**: list view shows truncated description as a secondary column; detail view shows full description as a paragraph + each source's resolved URL when present, falling back to the redirect URL when not.

## Live findings worth recording

- **81 / 81 prod google_news CBM tokens are modern opaque format.** The decoder returns `None` for all of them and emits `google_news.decode_failed reason=no_url_in_payload` to Cloud Run logs. This is *expected and documented* per D-Q3; the SPA falls back to the news.google.com redirect URL. If Google ever reverts to old-style tokens, the existing code path will start populating `resolved_url` automatically — no code change needed.
- **47 / 77 prod topics had a description recoverable from `raw_payload->>'summary'`.** Recovered live during T05 smoke against the docker postgres dev DB (which contains a 5-day-old snapshot of production data). The remaining 30 are HN-sourced (no summary in payload) or had empty summaries at first observation.
- **Pre-existing test failure in `services/crawler/tests/test_cross_source_dedup.py`** — `run_once` signature drift from Phase 3 (the test still passes `enabled_sources=…` kwarg, but the orchestrator signature changed). Logged to `.planning/phases/04.5-topic-description-capture-url-resolution/deferred-items.md`; **deselected** from Phase 4.5 regression runs; **not** caused by this plan. Schedule fix for the next maintenance plan.
