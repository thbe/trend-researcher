# Plan 02-02: RSS Source Adapters (Wave 2)

**Phase**: 2 — Multi-Source Ingest
**Mode**: mvp
**Depends on**: Plan 02-01

## Goal
Add a `RssSource` adapter that satisfies `SourcePort`, parameterized by feed URL. Register two instances: `nyt_homepage` and `google_news`. After this plan, `crawler run-once` fetches HN + 4 Reddit + 2 RSS = 7 sources in one run.

## Why one adapter, two instances
NYT and Google News are both standard RSS 2.0 feeds. One adapter handles both. Each gets its own `name` so attribution stays clean.

### T02 — RSS adapter (TDD: tests first)
- **Step 1 — write failing tests**: file `services/crawler/tests/test_rss_adapter.py`
  - Hand-rolled RSS 2.0 XML fixture string with 4 items: 3 valid + 1 missing `<title>`.
  - Mock `httpx.AsyncClient` with `httpx.MockTransport` returning the fixture as `text`.
  - Test `test_returns_three_items_skipping_broken`: 3 items, ranks 1/2/4 (gap preserved when item 3 is dropped).
  - Test `test_source_name_matches_constructor`: `RssSource(name="nyt_homepage", feed_url=...)` → items have `source_name == "nyt_homepage"`.
  - Test `test_raw_payload_is_json_serializable`: `json.dumps(item.raw_payload)` does not raise (no feedparser objects leaked).
  - Test `test_observed_at_is_fetch_time_not_pubdate`: `observed_at` is recent (within 5s of `datetime.now(timezone.utc)`), not the fixture's `<pubDate>` from 2020.
  - Test `test_user_agent_header_set`: outgoing request used the Trend-Researcher UA.
  - Run `uv run --package crawler pytest tests/test_rss_adapter.py -v` → all FAIL (RED).
- **Step 2 — implement to green**: file `services/crawler/src/crawler/adapters/sources/rss.py`
  - Class `RssSource(name: str, feed_url: str, http_client=None)`:
    - Same constructor pattern as Reddit/HN adapters.
    - `fetch(top_n)`:
      1. GET `feed_url` with the same UA as Reddit (10s timeout, `Accept: application/rss+xml, application/xml`).
      2. Parse via `feedparser.parse(response.text)` inside `await asyncio.to_thread(...)` (feedparser is sync).
      3. Iterate `parsed.entries[:top_n]` and build `RawItem` per entry:
         - `title` = `entry.title`
         - `url` = `entry.link`
         - `native_rank` = enumerate from 1
         - `observed_at` = wall clock at fetch time (NOT `entry.published_parsed`; pubDate goes into raw_payload)
         - `raw_payload` = `{"title": entry.title, "link": entry.link, "published": entry.get("published"), "summary": entry.get("summary")}` (don't dump the whole feedparser object — it's not JSON-serializable)
      4. Skip entries missing title or link (log + count, no rank renumbering).
- **Step 3 — verify GREEN**: `uv run --package crawler pytest tests/test_rss_adapter.py -v` → all 5 tests pass.
- Commit: `feat(02-02-T02): RssSource adapter (TDD: 5 tests written first, then impl)`
