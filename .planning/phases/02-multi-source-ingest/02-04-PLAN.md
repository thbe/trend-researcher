### T02 — Live E2E smoke (orchestrator-run, documented)
This is a runnable script + a documented procedure, NOT a pytest test (we don't want CI hitting the live internet).

- File: `scripts/smoke_phase2.sh` (new):
  ```bash
  #!/usr/bin/env bash
  set -euo pipefail
  echo "==> tearing down"
  docker compose down -v
  echo "==> bringing up postgres"
  docker compose up -d postgres
  # wait for healthy
  for i in {1..30}; do docker compose ps postgres | grep -q "healthy" && break; sleep 1; done
  echo "==> applying migrations"
  uv run --package core alembic upgrade head
  echo "==> first crawl"
  docker compose run --rm crawler run-once --top-n 30
  echo "==> per-source counts after first crawl"
  PGPASSWORD=trend psql -h localhost -U trend -d trend_researcher -c \
    "SELECT source_name, count(*) FROM topic_sources GROUP BY source_name ORDER BY 2 DESC;"
  echo "==> total topics + observation_count distribution"
  PGPASSWORD=trend psql -h localhost -U trend -d trend_researcher -c \
    "SELECT count(*) AS total_topics, sum(CASE WHEN observation_count >= 2 THEN 1 ELSE 0 END) AS multi_observation FROM topics;"
  echo "==> cross-source dedup spot check"
  PGPASSWORD=trend psql -h localhost -U trend -d trend_researcher -c \
    "SELECT t.id, t.title, count(DISTINCT ts.source_name) AS source_count
     FROM topics t JOIN topic_sources ts ON ts.topic_id = t.id
     GROUP BY t.id, t.title HAVING count(DISTINCT ts.source_name) >= 2
     ORDER BY source_count DESC LIMIT 10;"
  echo "==> second crawl (dedup proof)"
  docker compose run --rm crawler run-once --top-n 30
  echo "==> per-source counts after second crawl (S2 should be ≈ 2*S1)"
  PGPASSWORD=trend psql -h localhost -U trend -d trend_researcher -c \
    "SELECT source_name, count(*) FROM topic_sources GROUP BY source_name ORDER BY 2 DESC;"
  echo "==> teardown"
  docker compose down
  echo "==> done"
  ```
- `chmod +x scripts/smoke_phase2.sh`
- Run it. Capture full output to `.planning/phases/02-multi-source-ingest/SMOKE-RESULTS.md` with timestamp.
- **Pass criteria**:
  1. Both crawl invocations exit 0.
  2. After first crawl: ≥ 5 of 7 sources present in `topic_sources` (allows for one or two being rate-limited or down). HN and at least one Reddit and at least one RSS must be among the present.
  3. After first crawl: total topics is in `[100, 200]` (rough sanity — 30 per source × 7 = 210 max minus dedup).
  4. Cross-source dedup spot check returns ≥ 1 row (at least one story showed up on multiple sources). If 0, document as a soft pass (live news may not have any cross-publisher overlap that minute) and note the unit test in 02-03 already proves the mechanism.
  5. After second crawl: per-source `S2 ≥ S1` for every source that succeeded both times. `topics` row count delta ≤ 10% of S1 (most stories repeat; some new ones are normal).
  6. `failed_sources` log field, if non-empty, lists only sources we accept may be flaky (Reddit 429 most common); document any failures in SMOKE-RESULTS.md.
- Commit (after smoke green): `feat(02-04-T02): live E2E smoke script + recorded results for 7 sources`
