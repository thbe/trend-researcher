<action>
Append a new `## Multi-Tenant Platform (MT-*)` section to
`.planning/REQUIREMENTS.md` with the following requirements (each with a 1-2
sentence description matching the existing file's style):

- **MT-001** — Departments are first-class entities; the system supports N≥1
  departments coexisting with isolated configuration.
- **MT-002** — Users may belong to multiple departments; role is per
  `(user, department)` pair. Roles: `viewer`, `analyst`, `dept_lead`. System
  superadmin is a separate orthogonal flag.
- **MT-003** — Topics remain global (single deduplicated stream); all
  assessment artefacts are scoped to a department.
- **MT-004** — Crawler runs as a single global one-shot job whose effective
  source list is the union of all departments' active source subscriptions.
- **MT-005** — Assessment frameworks are pluggable; v1 seeds `verdict`,
  `swot`, `pestle`. Each department selects ≥1 enabled framework with one
  default.
- **MT-006** — Business cases are uniquely keyed on `(topic, department,
  framework, prompt_version, model_used)`; the structured framework-specific
  output lives in a JSONB column.
- **MT-007** — A harmonization view per topic shows side-by-side
  business_cases from every `(department, framework)` pair that assessed it,
  plus an optional admin-authored Net View free-text annotation
  (`topic_harmonizations`).
- **MT-008** — Existing single-tenant production data migrates losslessly
  into a seeded `Default` department; the existing seed user becomes
  `is_superadmin = true` and `dept_lead` of Default.
- **MT-009** — Ingest stays AI-free (ARC-001 preserved across the multi-tenant
  refactor); the crawler's only dependency on departments is reading the
  union source-subscription list.
</action>
