# Plan 10-05: Harmonization View + Phase 10 Closeout (Wave 5)

**Phase**: 10 — Multi-Tenant Market Intelligence Platform
**Mode**: mvp
**Depends on**: 10-04 (BusinessCaseCard + framework card components shipped;
session/role stores exist)
**REQ-IDs in scope**: MT-011 (cross-department side-by-side view), MT-012
(Net View annotation)
**Autonomous**: true — small migration, one new route, one new SPA tab.
Schema change is additive (new table only); no risk to existing data.

## Goal
Deliver Option B harmonization (G7) and close out Phase 10:
- New `topic_harmonizations` table: one optional row per topic holding a
  free-text "Net View" + author + timestamps.
- `GET /api/topics/{id}/harmonization` returns all business_cases for the
  topic (across all departments and frameworks the requester is allowed to
  see) + the meta annotation if present.
- `PUT /api/topics/{id}/harmonization` upserts the Net View (dept_lead+ or
  superadmin).
- TopicDetail gains a "Cross-department views" tab rendering one
  `BusinessCaseCard` per `(department, framework)` pair plus the Net View
  editor.
- Architecture docs refreshed for multi-tenant model.
- Phase 10 SUMMARY written; ROADMAP/STATE marked done.

## Slice shape (vertical)
Migration → ORM mirror → core test → new API router + schemas → API tests →
SPA tab + editor → SPA test → docs refresh → closeout.

## Threat model
- **Read visibility**: per CONTEXT.md "Open" question, decision here is —
  any logged-in user can READ the harmonization payload (including
  business_cases from depts they don't belong to). Rationale: the Net View
  is a cross-cutting org-wide signal; hiding individual dept assessments
  defeats the purpose. Write access stays restricted (dept_lead of ANY dept
  or superadmin). Documented in the route docstring. If business pushes
  back later, switching to "must be member of at least one assessing dept"
  is a one-line change.
- **Concurrent Net View edits**: two dept_leads editing simultaneously →
  last write wins. Acceptable for v1 (single-operator-per-org expected). No
  optimistic concurrency token. Documented in route comment.
- **Empty topic case**: a topic with zero business_cases (no dept has
  assessed yet) — harmonization endpoint returns
  `{business_cases: [], net_view: null}` cleanly; UI shows empty state.
- **Migration ordering**: 0019 only adds a table — no FK to anything that
  might still be evolving. Safe to ship.

  - Requires auth + (any of: `is_superadmin`, role `dept_lead` in any dept).

    insert only.)
  - 404 if topic not found.
  - Returns the same shape as GET.

- `DELETE /api/topics/{topic_id}/harmonization`:
  - Same write auth.
  - Removes the row. Idempotent (204 if absent).

Wire router in `services/api/src/api/routes/__init__.py` + `main.py`.
