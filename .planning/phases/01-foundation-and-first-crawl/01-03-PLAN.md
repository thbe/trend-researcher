<objective>
Define the pure domain types and port protocols for the crawler with ZERO I/O. After this plan, `services/crawler/src/crawler/domain/` and `services/crawler/src/crawler/ports/` contain everything needed for adapters (next wave) to slot in. The dedup function is fully unit-tested without touching Postgres or the network.

This plan implements the architectural spine: `RawItem` (input shape from any source), `Topic`-related domain types, `dedup_key()` + match function (rapidfuzz), `SourcePort`, `TopicRepositoryPort`. ING-009 (no AI in ingest) is enforced structurally — domain has zero LLM imports.
</objective>

<must_haves>
  <truths>
    - `domain/` modules MUST NOT import from `adapters/`, `app/`, `sqlalchemy`, `asyncpg`, `httpx`, or any I/O library
    - Dedup uses rapidfuzz `token_set_ratio` with threshold 85 (REQ ING-007 — formal threshold)
    - Ports are Python `Protocol` classes (PEP 544) — adapters are duck-typed against them
    - `SourcePort.fetch()` is async and returns `list[RawItem]`
    - `TopicRepositoryPort` exposes the operations the dedup-and-upsert flow needs: find-candidate, insert-new, update-existing
    - `RawItem` carries enough info for both fresh-insert and update paths: title, url, source_name, native_rank, observed_at, raw_payload
  </truths>
</must_haves>

    ```python
            """

    ```

    ```python
    from dataclasses import dataclass
    from datetime import datetime
    from typing import Protocol
    from uuid import UUID
    from crawler.domain.raw_item import RawItem

    @dataclass(frozen=True, slots=True)
    class TopicCandidate:
        """Lightweight projection of an existing Topic for dedup matching.

        We don't expose the full SQLAlchemy Topic model across the port boundary —
        the domain only needs id and title to decide 'same topic?'.
        """
        id: UUID
        title: str
        last_seen_at: datetime
        observation_count: int

    class TopicRepositoryPort(Protocol):
        async def find_candidates(self, dedup_key: str, limit: int = 50) -> list[TopicCandidate]:
            """Return existing topics whose normalized title shares tokens
            with the given dedup_key. Implementation may use trigram, ILIKE,
            or simply return recent topics — the dedup decision is made in
            the domain layer.
            """
            ...

        async def insert_new(self, item: RawItem) -> UUID:
            """Insert a new Topic and the first TopicSource row, return new topic id."""
            ...

        async def update_existing(self, topic_id: UUID, item: RawItem) -> None:
            """Append a new TopicSource row, bump last_seen_at, increment observation_count.
            If a TopicSource row with the same (source_name, url, observed_at) already
            exists, this is a no-op for that row (the unique constraint handles it).
            """
            ...
    ```
