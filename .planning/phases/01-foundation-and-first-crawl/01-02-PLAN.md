<objective>
Define the v1 Postgres schema in `packages/core` (the single source of schema truth for all services) and produce the first Alembic migration that creates `topics` and `topic_sources`. After this plan, applying the migration against a Postgres database creates exactly two tables with the correct columns, types, indexes, and foreign keys.

Stack is locked from CONTEXT/m0086: SQLAlchemy 2.x async + asyncpg + Alembic.
</objective>

<must_haves>
  <truths>
    - Schema is owned by `packages/core` ONLY — no other service ships its own migrations (locked architectural decision)
    - `topics` table holds one row per distinct topic (deduplicated). Re-crawls UPDATE existing rows
    - `topic_sources` is many-to-one against `topics` and accumulates per-observation references
    - Breadth and longevity are NOT stored — they are computed via SQL at query time (STO-005)
    - `topics.metadata` is a JSONB blob for source-agnostic extra fields (STO-001 + REQ shape from b3)
    - Alembic uses async engine (asyncpg)
  </truths>
</must_haves>

<task id="01-02-T02">
  <title>Define SQLAlchemy ORM models for topics and topic_sources</title>
  <read_first>

  </read_first>
  <action>
    Create `packages/core/src/core/models.py` defining a `Base = DeclarativeBase` and two model classes using SQLAlchemy 2.x typed Mapped/mapped_column syntax.

    **Topic model** maps table `topics` with columns: `id` (UUID primary key, server default `gen_random_uuid()`), `title` (Text, NOT NULL, indexed), `description` (Text, nullable), `first_seen_at` (TIMESTAMP WITH TIME ZONE, NOT NULL, server default `now()`), `last_seen_at` (TIMESTAMP WITH TIME ZONE, NOT NULL, server default `now()`, indexed), `observation_count` (Integer, NOT NULL, default 1), `topic_metadata` (JSONB, NOT NULL, server default `'{}'::jsonb`), `created_at` (TIMESTAMP WITH TIME ZONE, NOT NULL, server default `now()`), `updated_at` (TIMESTAMP WITH TIME ZONE, NOT NULL, server default `now()`). Note the Python attribute is `topic_metadata` mapped to the SQL column `metadata` — `metadata` is a reserved SQLAlchemy attribute on `DeclarativeBase` and cannot be used as an attribute name. Map via `mapped_column("metadata", JSONB, ...)`.

    **TopicSource model** maps table `topic_sources` with columns: `id` (UUID primary key, server default `gen_random_uuid()`), `topic_id` (UUID, NOT NULL, foreign key to `topics.id` ON DELETE CASCADE, indexed), `source_name` (Text, NOT NULL, indexed — values like `"hackernews"`, `"reddit:r/all"`, `"nyt-rss"`), `url` (Text, NOT NULL), `native_rank` (Integer, nullable — the source's own ranking position), `observed_at` (TIMESTAMP WITH TIME ZONE, NOT NULL, server default `now()`, indexed), `raw_payload` (JSONB, nullable). Add a unique constraint on the tuple `(topic_id, source_name, url, observed_at)` to prevent exact-duplicate observations within the same crawl.

    Use `from sqlalchemy.dialects.postgresql import UUID, JSONB`. Use `from sqlalchemy import ForeignKey, UniqueConstraint, Index, text`. Use `from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column, relationship`. Add a `topic.sources: Mapped[list["TopicSource"]] = relationship(...)` and the back-populates side on `TopicSource`.
  </action>
  <acceptance_criteria>
    - File `packages/core/src/core/models.py` exists
    - File contains `class Topic(Base)` and `class TopicSource(Base)`
    - File contains `__tablename__ = "topics"` and `__tablename__ = "topic_sources"`
    - File contains `mapped_column("metadata", JSONB` (SQL column literally named `metadata`, Python attribute named `topic_metadata`)
    - File contains `ForeignKey("topics.id"` with `ondelete="CASCADE"`
    - File contains `UniqueConstraint("topic_id", "source_name", "url", "observed_at"`
