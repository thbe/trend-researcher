<objective>
Stand up the monorepo file layout and Python toolchain so all subsequent plans have a place to write code. After this plan, `uv sync` from the repo root resolves all four packages into a single virtualenv, and the directory shape matches the locked architectural decision (services/{crawler,api,assessor}/ + packages/core/ + web/).

This plan writes ZERO runtime code. It only creates the skeleton: empty `__init__.py` files, per-service `pyproject.toml` stubs, and the workspace root that ties them together.
</objective>

<must_haves>
  <truths>
    - Monorepo uses uv workspace (locked decision, see CONTEXT/m0086)
    - Single `uv.lock` at repo root spans all workspace members
    - Only `services/crawler/` and `packages/core/` will get real code in Phase 1; `services/api/`, `services/assessor/`, `web/` are scaffold-only
    - ARC-001: AI/LLM code MUST NOT appear anywhere under `services/crawler/` or `packages/core/`
    - ARC-003: each service follows ports & adapters layout (folders created here, code in later plans)
  </truths>
</must_haves>
