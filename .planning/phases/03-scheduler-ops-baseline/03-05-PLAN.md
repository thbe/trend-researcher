# Plan 03-05: Smoke + Phase 3 closeout (Wave 5)

**Phase**: 3 — Scheduler & Ops Baseline
**Mode**: mvp
**Depends on**: Plan 03-04 (full stack must be wired in compose)
**REQ-IDs in scope**: success criteria #5 (24h+ unattended verification — phase exit gate via smoke + post-close real-cron observation)

## Goal
Prove the Phase 3 wiring end-to-end with a runnable smoke script, document the real-cron observation procedure, update README operator section, bump STATE + ROADMAP, push to remote. Phase 3 closes here.

## Slice shape
operator-facing script + docs + state + ship.

## Why `autonomous: false`
The smoke script hits the live internet (3 real crawls of HN + NYT + Google News) AND requires a `git push` at the end. Both are non-autonomous operator gates — same posture as Plan 02-04 close.

## Locked design decisions
- **Smoke = 3 manual triggers** (not 24h wall-clock wait) — closes phase exit gate per CONTEXT.md §D.
- **Post-close observation** = real-cron 24-48h check, recorded as a one-line entry in PoC observation log (separate from this phase plan; happens after Phase 3 closes). If the real cron fails, opens a Phase 3.1 hot-fix.

## Tasks

### T01 — `scripts/smoke_phase3.sh`
- **read_first**: `scripts/smoke_phase2.sh` (existing pattern), Plan 03-04 (compose service names: `postgres`, `api`, `scheduler`, `crawler`), Plan 03-01 (the `crawl_runs` shape), Plan 03-02 (`/healthz`, `/runs?limit=`).
- **action**: New file `scripts/smoke_phase3.sh`. Same `set -euo pipefail` + `PSQL=` pattern as Phase 2 smoke. Steps (numbered):
  1. `docker compose down -v` (clean slate).
  2. `docker compose build crawler api scheduler` (rebuild all images to pick up Phase 3 code).
  3. `docker compose up -d postgres` + healthy-wait loop (copy from smoke_phase2).
  4. Apply migrations from host: `uv run --package core alembic -c packages/core/alembic.ini upgrade head` (gets 0001 + 0002).
  5. `docker compose up -d api scheduler` + 5s sleep for startup.
  6. Sanity: `curl -fs localhost:8000/healthz | grep -q '"status":"ok"'` (script `set -e` hard-fails the smoke on non-200 or missing field).
  7. Verify scheduler crontab loaded: `docker logs trend-scheduler 2>&1 | grep -q "starting crond"`.
  8. **First trigger**: `docker compose run --rm crawler run-once --top-n 30`. Confirm `crawl_runs` row count: `$PSQL -c "SELECT count(*) FROM crawl_runs;"` → expect `1`.
  9. **Second trigger**: same. Expect `2`.
  10. **Third trigger**: same. Expect `3`.
  11. Verify observation_count advances on persistent topics: `$PSQL -c "SELECT count(*) FROM topics WHERE observation_count >= 2;"` → expect ≥ 1 (HN + NYT + GoogleNews are stable enough across 3 immediate runs that at least one repeat is essentially guaranteed; soft-pass criterion logged either way).
  12. Verify `/runs` returns the 3 rows newest-first: `curl -fs "localhost:8000/runs?limit=5" | jq '.runs | length'` → expect `3`.
  13. Verify `failed_sources` discipline: `$PSQL -c "SELECT failed_sources FROM crawl_runs ORDER BY started_at DESC LIMIT 3;"` — print for inspection (not asserted; the operator may legitimately see Reddit-style transient failures from upstream).
  14. Print the per-run summary table: `$PSQL -c "SELECT started_at, totals_inserted, totals_updated, totals_errors, failed_sources FROM crawl_runs ORDER BY started_at;"`.
  15. Print disabled-source verification hint: `echo "==> to verify CRAWLER_DISABLED_SOURCES, run: CRAWLER_DISABLED_SOURCES=hackernews docker compose run --rm crawler run-once --top-n 5"`.
  16. `docker compose down` (keep volume — operator may want to inspect post-mortem).
  17. `echo "==> Phase 3 smoke PASS"` if `set -e` survived to here.
- Make executable: `chmod +x scripts/smoke_phase3.sh`.
- **acceptance_criteria**:
  - `bash -n scripts/smoke_phase3.sh` (syntax check) exits 0.
  - `./scripts/smoke_phase3.sh` runs end-to-end and emits `Phase 3 smoke PASS` (live network, manual operator run).
  - `crawl_runs` count goes 0 → 1 → 2 → 3 across the three triggers.
  - `/healthz` returned 200.
  - `/runs?limit=5` returned 3 rows.
- **commit**: `test(03-05-T01): scripts/smoke_phase3.sh — 3-trigger E2E smoke`
