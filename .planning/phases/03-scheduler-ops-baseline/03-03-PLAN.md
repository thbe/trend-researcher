# Plan 03-03: Source-disable via `CRAWLER_DISABLED_SOURCES` env (Wave 3)

**Phase**: 3 — Scheduler & Ops Baseline
**Mode**: mvp
**Depends on**: Plan 03-01 (uses Phase 2 source-list mechanics; no code from 03-02)
**REQ-IDs in scope**: OPS-003

## Goal
Let the operator disable any source by listing its name in a `CRAWLER_DISABLED_SOURCES` env var (csv). The next `docker compose run --rm crawler` reads the var, filters its source list, and crawls only the rest. No rebuild, no redeploy, no UI — Phase 5 owns UI-driven config.

## Slice shape
Tiny — env var → composition filter → unit tests. Runs in parallel-after-03-01 because it touches `composition.py` only and 03-02 doesn't touch the crawler at all. Sequenced as Wave 3 (after 03-02) only because we want one wave per plan to keep commits ordered with the scheduler observation; no actual file conflict.

## Threat model
- Env var read at container startup. Worst case if mis-set: a source the operator wanted is silently skipped. Mitigation: log the parsed disabled set at startup so `docker logs` shows the active filter.
- No code injection vector (the value is split on `,` and used only as set-membership checks against statically-known source names).

## Locked design decisions
- **Variable name**: `CRAWLER_DISABLED_SOURCES` (matches the `CRAWLER_*` prefix convention already in compose).
- **Format**: comma-separated, whitespace-trimmed, case-insensitive comparison against `source.name`. Empty/unset = no filtering.
- **Unknown names**: logged as a warning (`crawler.disabled_sources.unknown`, with the unknown set), do NOT raise. Rationale: makes operator typos visible without breaking the run; Phase 5's UI catches the typo properly.
- **Empty result**: if the filter eliminates ALL sources, `build_sources()` returns `[]` and the orchestrator runs to completion writing a `crawl_runs` row with all zeros and `failed_sources=[]`. No error — disabling all sources is a valid (if pointless) operator choice.

## Pattern map
- `services/crawler/src/crawler/app/composition.py` already builds and returns the source list. Add the filter after the existing list construction, before the return.
- Test pattern: pure unit tests on `build_sources()` (no Postgres, no network) — pytest with `monkeypatch.setenv` / `monkeypatch.delenv`.

## Tasks

### T01 — Add env-driven filter to `build_sources()`
- **read_first**: `services/crawler/src/crawler/app/composition.py`.
- **action**: Add a module-level helper `_parse_disabled_sources(raw: str | None) -> set[str]` that returns an empty set for None/empty, otherwise splits on `,`, strips whitespace, lowercases each entry, returns a set. In `build_sources()`, after constructing the full source list:
  - Read `os.environ.get("CRAWLER_DISABLED_SOURCES")`.
  - Compute `disabled = _parse_disabled_sources(raw)`.
  - Compute `unknown = disabled - {s.name.lower() for s in sources}`.
  - If `unknown`: `_log.warning("crawler.disabled_sources.unknown", unknown=sorted(unknown))` (add a module-level structlog logger if not present).
  - If `disabled`: log `_log.info("crawler.disabled_sources.applied", disabled=sorted(disabled))`.
  - Return `[s for s in sources if s.name.lower() not in disabled]`.
- Update the docstring of `build_sources` to mention the env var.
- **acceptance_criteria**:
  - `CRAWLER_DISABLED_SOURCES=hackernews uv run --package crawler crawler run-once --top-n 1` runs and produces a `crawl_runs` row whose `per_source` has no `hackernews` key (only NYT + Google News) — manual verification.
  - With env unset, `build_sources()` returns 3 sources (HN, NYT, Google News).
- **commit**: `feat(03-03-T01): build_sources filters by CRAWLER_DISABLED_SOURCES env`

### T02 — Unit tests
- **read_first**: T01 output, existing `services/crawler/tests/test_*.py` patterns (look for monkeypatch usage on env).
- **action**: New file `services/crawler/tests/test_composition_disabled_sources.py` with these tests:
  - `test_no_env_returns_all_sources(monkeypatch)`: `monkeypatch.delenv("CRAWLER_DISABLED_SOURCES", raising=False)`. Asserts `len(build_sources()) == 3` and `{s.name for s in build_sources()} == {"hackernews", "nyt_homepage", "google_news"}`.
  - `test_disable_one(monkeypatch)`: `monkeypatch.setenv("CRAWLER_DISABLED_SOURCES", "hackernews")`. Asserts hackernews is absent, NYT and Google News present.
  - `test_disable_multiple_csv(monkeypatch)`: sets `"hackernews, nyt_homepage"`. Asserts only `google_news` remains.
  - `test_disable_case_insensitive(monkeypatch)`: sets `"HackerNews"`. Asserts hackernews absent.
  - `test_disable_whitespace_tolerated(monkeypatch)`: sets `"  hackernews  ,  nyt_homepage  "`. Asserts only google_news remains.
  - `test_unknown_source_warns_but_doesnt_raise(monkeypatch, caplog)`: sets `"reddit,nonexistent"`. Asserts `len(build_sources()) == 3` (no real source matched), and `caplog` (or structlog testing harness) shows the `crawler.disabled_sources.unknown` warning containing `nonexistent`.
  - `test_disable_all(monkeypatch)`: sets `"hackernews,nyt_homepage,google_news"`. Asserts `build_sources() == []`.
  - `test_empty_string_treated_as_unset(monkeypatch)`: sets `""`. Asserts all 3 sources returned.
- **acceptance_criteria**:
  - `uv run --package crawler pytest tests/test_composition_disabled_sources.py -v` shows 7 pass.
  - Tests are pure unit (no DB, no network).
- **commit**: `test(03-03-T02): CRAWLER_DISABLED_SOURCES filter — 7 unit tests`
